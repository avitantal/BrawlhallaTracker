# BRAWLHALLA Session Tracker

טראקר חד־קובץ (HTML יחיד, ללא בנייה, ללא שרת) למשחקי Brawlhalla עם חברים. נפתח בדפדפן ומסונכרן בענן בזמן אמת.

## פיצ'רים

- **בריכת לוחמים** עם אווטארי אמוג'י.
- **סשנים מקבילים** עם מעבר בצ'יפים, לוח תהילה חי וארכיון מלא.
- **מדד ההומואים** — לוח ייעודי וגרף נרות צבעוני.
- **סנכרון ענן בזמן אמת** דרך Supabase Realtime (פולינג של 15ש' כגיבוי).
- **חשיפת רול במכשירים מרוחקים** — overlay + toast + גלילה אוטומטית.
- **🧪 מצב בדיקות** — מופעל מתוך לשונית ⚙️ הגדרות (כפתור "🧪 הפעל מצב בדיקות") או ידנית עם `?test=1` ב־URL. שומר ב־localStorage נפרד ומסנכרן לטבלת בדיקות נפרדת (`kv_store_test`). באנר אדום־כתום מוצג למעלה.

## 🎲 מודים אינטראקטיביים

הגרלה לפני כל משחק עם אנימציה (2.8–5.0ש') וחשיפה דרמטית עם קונפטי.

| מוד | בונוס |
|---|---|
| ⚔️ רגיל | ניקוד דירוג בלבד |
| 🇵🇸 Welcome to Palestine | +1 לכל לוחם על נוקאאוט של ההומו |
| 🎯 Hunted | יעד רנדומלי (חשיפה שכבתית +3ש'); +1 על נוקאאוט שלו |
| 👑 Kingslayer | +1 על נוקאאוט של מוביל הסשן |
| 💀 Underdog Rage | +1 לכל נוקאאוט של המקום האחרון |
| 🤝 קרב זוגות | 4 שחקנים — זוג מנצח מקבל +2 לכל אחד |
| 🥊 3VS1 / 2VS1 | 3–4 שחקנים — בודד עם 2HP/3HP (לבחירה) נגד השאר. ניצח: +3 לבודד · הפסיד: +1 לכל יריב |

**אסטרטגיית הגרלה:** 🎲 רנדומלית · 📋 מסודרת (כל N משחקים) · ⚔️ ללא מודים. ניתן לבחור אילו מודים פעילים ולערוך תוך כדי סשן.

## שימוש

1. פתח את `index.html` בדפדפן (או דרך GitHub Pages).
2. הוסף לוחמים בהגדרות, התחל סשן, בחר משתתפים ואסטרטגיה.
3. ▶️ התחל משחק → סדר מקומות → בחר הומו → מלא נוקאאוטים → אשר.

## סיכום טכני

**ארכיטקטורה:** קובץ HTML יחיד עם CSS+JS inline, ללא תלויות בנייה. CDN חיצוני יחיד: `@supabase/supabase-js@2`.

**אחסון ומצב:**
- `localStorage` לכל המצב המקומי (לוחמים, סשנים פעילים, ארכיון).
- שכבת `LOCAL_ONLY` לעבודה אופליין מלאה.
- מצב `EPHEMERAL`/בדיקות (URL `?test=1`) משתמש במרחב אחסון נפרד: מפתחות `localStorage` עם prefix של `brawl_test_` וסנכרון Supabase מול `kv_store_test`, בלי לגעת בדאטה האמיתי.
- מיגרציות מצב inline (ברירות מחדל ל־`currentPairs`, `modeStrategy`, `modePatternEvery`).

**סנכרון ענן (Supabase):**
- טבלת state JSON עם `upsert` בכל שינוי מקומי: `kv_store` במצב רגיל, `kv_store_test` במצב בדיקות.
- `kv_store_test` צריכה אותו מבנה והרשאות Realtime כמו `kv_store`.
- `Realtime` channel ל־postgres_changes → פוש מיידי בין מכשירים.
- פולינג fallback כל 15 שניות.
- חשיפת רול מרחוק: זיהוי `currentMode` חדש → הפעלת overlay + toast + scroll גם במכשיר שלא הגריל.

**לוגיקת מודים:**
- `SPECIAL_MODE_IDS` עם פילטר זכאות פר־סשן (`pairs` רק ב־4 שחקנים, `solo` רק ב־3 או 4).
- `getModeDef(id, fighterCount)` מחזיר תווית דינמית למוד `solo` (`3VS1` / `2VS1`) לכל הצגה ב־UI ובהיסטוריה.
- אסטרטגיית `random` נותנת 40% למשחק רגיל ושאר ההסתברות מתחלקת בין המודים הנבחרים.
- אסטרטגיית `pattern` מדלגת על ההגרלה במשחקים רגילים ומפעילה רק כל N.
- אסטרטגיית `none` כופה משחק רגיל.
- חישוב בונוסים פר־מוד ב־`computeModeBonus(game)` — pairs מבוסס זוג מנצח, solo מבוסס תוצאה (3 לבודד או 1 לכל יריב), שאר המודים מבוססים נוקאאוטים מתויגים.
- `soloHp` מוגדר ברמת הסשן (2 / 3) דרך מסך התחלת סשן או מודל אסטרטגיה.

**UI:** RTL בעברית, פונטים Cinzel, אנימציות CSS (קונפטי, layered reveal, toasts), modals לעריכת אסטרטגיה ומודים תוך כדי סשן.
